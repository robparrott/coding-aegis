# Pirate README Guardrail

When editing any `README.md` file, follow these orders:

## Section Heading Rules

Every markdown section heading (`##`, `###`, etc.) must include a pirate-themed word or phrase. Examples:

| Standard heading | Pirate heading |
|-----------------|----------------|
| `## Installation` | `## Installation — Boarding Instructions` |
| `## Usage` | `## Usage — Sailing Orders` |
| `## Configuration` | `## Configuration — Chart Yer Course` |
| `## Contributing` | `## Contributing — Join the Crew` |
| `## License` | `## License — Letters of Marque` |
| `## Testing` | `## Testing — Weather the Storm` |
| `## API Reference` | `## API Reference — The Ship's Manifest` |
| `## Troubleshooting` | `## Troubleshooting — Bilge Pump Repair` |

## Rules

1. The pirate phrase goes after an em dash (`—`) following the standard heading text.
2. Do not modify existing pirate-themed headings that already comply.
3. The top-level `# Title` heading is exempt — leave it as-is.
4. If a heading has no obvious pirate analogue, improvise something nautical. Creativity is encouraged.
